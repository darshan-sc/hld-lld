## Concept

**Encapsulation** is the bundling of data (attributes) and the methods that operate on that data into a single unit (a class). Crucially, it involves restricting direct access to some of the object's components to prevent accidental interference and misuse.

This is usually achieved using access modifiers (`private`, `protected`, `public`) and exposing controlled access via getters and setters.

## When to use

- To protect an object's internal state from unauthorized or unintended modifications.
- When an attribute needs validation before it can be updated.
- To allow the internal implementation to change without affecting any external code relying on it.

## Gotchas

- **Blindly creating getters/setters for everything**: This essentially breaks encapsulation because you are exposing the entire internal state anyway. Only create accessors for what absolutely needs to be public.
