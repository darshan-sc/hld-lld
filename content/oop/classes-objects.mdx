## Concept

**Classes** are blueprints or templates that define the structure and behavior (properties and methods) of a certain type of entity. 
**Objects** are concrete, realized instances built from those blueprints, each with its own independent state.

If a class is the blueprint for a house, the object is the actual physical house built at a specific address.

## When to use

- At the foundation of any Object-Oriented system.
- When you want to bundle data (attributes) and the functions that operate on that data (methods) into a single logical unit.
- To create multiple distinct entities that share the same structure but hold different values.

class BankAccount {
  ownerName: string;
  balance: number;

  constructor(ownerName: string, initialBalance: number) {
    this.ownerName = ownerName;
    this.balance = initialBalance;
  }

  deposit(amount: number): void {

## Gotchas

- **God Classes**: Creating a massive class that does everything instead of breaking it down under the Single Responsibility Principle.
- **Data Classes**: Creating classes that only contain getters/setters and no real behavior (an anti-pattern known as Anemic Domain Model).
