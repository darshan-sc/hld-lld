## Concept

**Inheritance** is a mechanism where a new class (subclass or derived class) derives properties and behaviors (methods) from an existing class (base or superclass). This represents an **IS-A** relationship.

It promotes code reusability and establishes a natural hierarchy between entities.

## When to use

- When you have a clear "IS-A" relationship (e.g., a Dog IS-A Animal, a Manager IS-An Employee).
- When you want to share common state and behavior among multiple derived classes to avoid duplication.
- *Caution*: Default to **Composition over Inheritance**. Inheritance creates tight coupling and a rigid hierarchy that can be hard to change later.

## Gotchas

- **Fragile Base Class problem**: A change in the superclass can inadvertently break subclasses.
- **Deep Inheritance Trees**: Having 5+ layers of inheritance makes the code incredibly hard to follow and debug.
- **Inheriting for code reuse only**: If Class B needs some code from Class A, but B is *not* an A (e.g., Stack inheriting from ArrayList), you should use Composition instead.
