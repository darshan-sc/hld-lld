## Concept

**Abstraction** is the principle of hiding the complex reality while exposing only the necessary parts. It allows the user to interact with a system using a simple interface, without needing to understand the intricate details of how it works underneath.

Think of a car: you know how to use the steering wheel and pedals (the interface), but you don't need to know how the combustion engine and transmission work (the implementation).

## When to use

- When you have complex logic that clients don't need to be aware of.
- When you want to separate *what* an object does from *how* it does it.
- When you need to provide a unified interface for different concrete implementations.

interface CoffeeMachine {
  brewCoffee(): string;
}

// Concrete implementation hiding complex internal steps
class BasicCoffeeMachine implements CoffeeMachine {
  private boilWater(): void { console.log("Boiling water..."); }

    this.grindBeans();

  }
}

const machine: CoffeeMachine = new BasicCoffeeMachine();
// The client only cares about brewing coffee

## Gotchas

- **Leaky Abstractions**: When internal details "leak" through the interface (e.g., exposing network errors in a generic logging repository).
- **Premature Abstraction**: Creating interfaces and base classes for concepts that only have one implementation, leading to unnecessary complexity.
