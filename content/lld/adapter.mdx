## Concept

The **Adapter** is a structural design pattern that allows objects with incompatible interfaces to collaborate.

It catches calls for one object and transforms them to format and interface recognizable by the second object. Think of it like a physical power adapter that allows a European plug to fit into a US wall outlet.

## When to use

- When you want to use some existing class, but its interface isn't compatible with the rest of your code.
- When you want to reuse several existing subclasses that lack some common functionality that can't be added to the superclass.
- When integrating a 3rd party library, legacy code, or an external API into a modern codebase.

## Gotchas

- **Adapter vs Decorator vs Proxy**: An Adapter changes the interface of an existing object. A Decorator enhances another object without changing its interface. A Proxy provides the same interface as the object it wraps.
- **Over-engineering**: Sometimes it's simpler just to change the `Adaptee`'s code so that it matches your interface, rather than writing a whole new Adapter class, provided you own the `Adaptee` source code.
