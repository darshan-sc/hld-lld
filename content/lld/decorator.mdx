## Concept

The **Decorator** is a structural design pattern that lets you attach new behaviors to objects by placing these objects inside special wrapper objects that contain the behaviors.

Because the wrapper implements the exact same interface as the wrapped object, you can stack as many wrappers as you want, dynamically altering an object's behavior at runtime without changing its code.

## When to use

- When you need to add extra behaviors to objects at runtime without breaking the code that uses these objects.
- When extending a class using inheritance is awkward, impossible, or leads to a combinatorial explosion of subclasses (e.g., `DarkScrollableAnimatedWindow`).
- A canonical example is Java's `InputStream` (e.g., `new BufferedInputStream(new FileInputStream(file))`).

## Code

<CodeBlock
  typescript={`// 1. The Component Interface
interface Notifier {
  send(message: string): void;
}

// 2. Concrete Component (The base behavior)
class EmailNotifier implements Notifier {
  send(message: string): void {
    console.log(\`Sending Email: \${message}\`);
  }
}

// 3. Base Decorator (Delegates work to the wrapped component)
abstract class NotifierDecorator implements Notifier {
  constructor(protected wrappee: Notifier) {}

  send(message: string): void {
    this.wrappee.send(message);
  }
}

// 4. Concrete Decorators (Add extra behavior before/after delegation)
class SMSDecorator extends NotifierDecorator {
  send(message: string): void {
    super.send(message); // Forward to wrapped object
    console.log(\`Sending SMS: \${message}\`);
  }
}

class SlackDecorator extends NotifierDecorator {
  send(message: string): void {
    super.send(message);
    console.log(\`Sending Slack ping: \${message}\`);
  }
}

// 5. Client Code
const stack1 = new EmailNotifier();
const stack2 = new SMSDecorator(stack1);
const stack3 = new SlackDecorator(stack2);

// Client triggers the outermost wrapper, which cascades down
stack3.send("Server is down!");`}

  void send(String message);
}

class EmailNotifier implements Notifier {
  public void send(String message) {
    System.out.println("Sending Email: " + message);
  }
}

abstract class NotifierDecorator implements Notifier {
  protected Notifier wrappee;
  public NotifierDecorator(Notifier wrappee) {
    this.wrappee = wrappee;
  }
  public void send(String message) {
    wrappee.send(message);
  }
}

class SMSDecorator extends NotifierDecorator {
  public SMSDecorator(Notifier wrappee) { super(wrappee); }
  @Override
  public void send(String message) {
    super.send(message);
    System.out.println("Sending SMS: " + message);
  }
}

class SlackDecorator extends NotifierDecorator {
  public SlackDecorator(Notifier wrappee) { super(wrappee); }
  @Override
  public void send(String message) {
    super.send(message);
    System.out.println("Sending Slack ping: " + message);
  }
}

public class Main {
  public static void main(String[] args) {
    Notifier stack = new SlackDecorator(new SMSDecorator(new EmailNotifier()));
    stack.send("Server is down!");
  }
}`}
  python={`from abc import ABC, abstractmethod

# Component Interface
class Notifier(ABC):
    @abstractmethod
    def send(self, message: str) -> None:
        pass

# Concrete Component
class EmailNotifier(Notifier):
    def send(self, message: str) -> None:
        print(f"Sending Email: {message}")

# Base Decorator
class NotifierDecorator(Notifier):
    def __init__(self, wrappee: Notifier):
        self._wrappee = wrappee

    def send(self, message: str) -> None:
        self._wrappee.send(message)

# Concrete Decorators
class SMSDecorator(NotifierDecorator):
    def send(self, message: str) -> None:
        super().send(message)
        print(f"Sending SMS: {message}")

class SlackDecorator(NotifierDecorator):
    def send(self, message: str) -> None:
        super().send(message)
        print(f"Sending Slack ping: {message}")

# Client Code
stack = SlackDecorator(SMSDecorator(EmailNotifier()))
stack.send("Server is down!")`}

## Gotchas

- **Initialization Hell**: Initializing a deeply nested decorated object can be verbose and look ugly (`new A(new B(new C(new D())))`). This is usually mitigated by combining the Decorator with a Builder or a Factory.
- **Order Dependency**: The order in which you wrap the decorators can sometimes affect behavior in unexpected ways. Design your decorators so their order doesn't matter, if possible.
