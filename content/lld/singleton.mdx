## Concept

**Singleton** is a creational design pattern that ensures a class has only one single instance throughout the application lifecycle, while providing a global access point to this instance.

## When to use

- When a class in your program should have just a single instance available to all clients (e.g., a shared Database connection pool, an application Settings/Configuration object, or a Logger).
- When you need stricter control over global variables. The Singleton guarantees that no other code will overwrite the instance.

## Gotchas

- **It is functionally a Global Variable**: Singletons introduce global state into an application, which makes unit testing incredibly difficult because state carries over between tests.
- **Multithreading issues**: In multithreaded environments (like Java/C#), multiple threads calling `getInstance()` simultaneously for the first time might accidentally create multiple instances. You must use synchronization or locks.
- **Dependency Injection**: Modern frameworks usually replace the Singleton pattern with Dependency Injection containers that manage a single instance for you, abstracting away the private constructor logic.
