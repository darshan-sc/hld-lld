## Concept

Delegate behavior to state objects as state transitions occur. Keep the implementation minimal, but the boundaries explicit.

## When to use

- When you want a clear separation of concerns in object interactions.
- When interview discussions require trade-offs, not just syntax.
- When you need an example that maps quickly to production design decisions.

  java={`interface State {
  String execute(String input);

class StateImpl implements State {

    return input.trim();
  }
}

  python={`class State:
  def execute(self, input: str) -> str:
    raise NotImplementedError

class StateImpl(State):
  def execute(self, input: str) -> str:
    return input.strip()

demo: State = StateImpl()
print(demo.execute("sample"))`}

## Gotchas

- Copying the pattern mechanically without checking the problem constraints.
- Ignoring object responsibilities, which usually leads to god classes.
