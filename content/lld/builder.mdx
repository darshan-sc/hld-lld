## Concept

The **Builder** is a creational design pattern that lets you construct complex objects step by step. The pattern allows you to produce different types and representations of an object using the same construction code.

Instead of having a massive constructor with a dozen optional parameters (the Telescoping Constructor anti-pattern), you extract the construction logic into separate *builder* objects.

## When to use

- To get rid of a "telescoping constructor" (`new Pizza(size, crust, null, true, false, null)`).
- When you want your code to be able to create different representations of some product (e.g., stone houses vs. wooden houses).
- To construct Composite trees or other complex objects in a step-by-step fashion.

## Gotchas

- **Overkill logic**: You don't always need a separate `Director` class (which manages the step sequence). Often, the client chaining builder methods is completely sufficient and more readable.
- **Mutability concerns**: Builders often carry mutable state while constructing. They must correctly `reset()` their state after `getResult()` to ensure they can be safely re-used.
