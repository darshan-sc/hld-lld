## Concept

Model board transitions, turns, and deterministic game logic. Keep the implementation minimal, but the boundaries explicit.

## When to use

- When you want a clear separation of concerns in object interactions.
- When interview discussions require trade-offs, not just syntax.
- When you need an example that maps quickly to production design decisions.

## Requirements

- Clarify functional flows and error paths before class design.
- Identify scale assumptions and consistency requirements up front.

## Entities / Classes

- SnakeLadderController: orchestration and request handling.
- SnakeLadderService: core business logic.
- SnakeLadderRepository: persistence boundary.

## Key Methods

- create() or handle() for command-style workflows.
- validate() for business and state checks.
- notify() for observer/event style side effects.

## Gotchas

- Starting with classes before extracting requirements and invariants.
- Mixing orchestration concerns with domain entities.
